# ICP Scoring Interactive Report — Design Spec

**Date:** 2026-03-15
**Status:** Approved
**Route:** `/admin/icp-scoring`
**Audience:** Leadership/strategy (top sections tell the story) + Sales (bottom sections are the workbench)

## Overview

Replaces the current table-only ICP scoring page with a scrolling interactive report. 7 sections that progress from methodology explanation → portfolio analysis → actionable prospect lists → full data explorer.

Data source: `/public/district_scores.json` (17,910 pre-scored districts, generated by `.tmp/run_scoring.py`).

## Architecture

Single client component page at `src/app/admin/icp-scoring/page.tsx`. No API routes — all data loaded from static JSON via `fetch` on mount. Client-side filtering, sorting, and pagination.

### Dependencies

- **Recharts 3** (already installed) — histogram, horizontal bar charts
- **Lucide React** (already installed) — section icons
- All styling via Tailwind with Fullmind design tokens from `Documentation/UI Framework/tokens.md`

### Data Loading & States

The JSON file (`district_scores.json`) is ~25MB raw but served gzip-compressed by Next.js (~3-5MB over the wire). Data loading is handled at the page level:

- **Loading:** Full-page centered skeleton with the sticky header visible. A single `animate-pulse` block with 3-4 placeholder card shapes. No individual section skeletons — all 7 sections render together once data arrives.
- **Error:** Full-page centered error state (same pattern as DataGrid: alert icon + "Something went wrong" + retry button) in place of all sections.
- **Empty:** Not expected (static file), but if JSON returns an empty array, show "No scoring data available" centered message.

The `*_details` JSON string fields (fit_details, value_details, etc.) are only parsed when a user expands a row in Section 7. No upfront parsing cost.

### Page Layout

- Full-page vertical scroll (not viewport-constrained)
- `bg-off-white` (`#FFFCFA`) page background
- Sticky header with page title + Export CSV button
- 7 content sections, each a `max-w-[1600px] mx-auto` card
- Sections separated by `gap-6` vertical spacing
- Cards: `bg-white rounded-lg border border-[#D4CFE2] shadow-sm p-6`

## Section 2 — Tier Distribution

**Purpose:** Show how the 17,910 districts break into 4 tiers.

### Stacked bar
- Full-width horizontal bar showing T1/T2/T3/T4 as proportional segments
- Colors: T1 `#F37167` (Coral), T2 `#D4A84B` (Warning), T3 `#8A80A8` (Secondary), T4 `#D4CFE2` (Border Default)

### Tier detail cards (4-column grid below the bar)
Each card shows for one tier:
- Tier label + color dot
- District count + % of total
- Customers vs Prospects (two numbers)
- Total vendor revenue

- Card style: same as stat cards but with a left border accent matching tier color (`border-l-3`)

## Section 3 — Score Distribution

**Purpose:** Visualize how scores are distributed — is it a bell curve? Top-heavy? Bimodal?

### Recharts BarChart
- X-axis: composite score in 5-point buckets (0-4, 5-9, ..., 95-100)
- Y-axis: district count
- Bar fill colored by composite score range (not tier assignment — a district at composite 65 may be Tier 2 if Fit or Value < 60, but its histogram bar is still coral): ≥60 coral, 40-59 gold, 20-39 plum-light, <20 muted
- Tooltip on hover: "Score 60-64: 87 districts (23 customers)"
- Chart height: 200px
- Recharts config:
  - `<BarChart>` with `<Bar>` using `fill` from a function that maps score range → tier color
  - `<XAxis>` with `text-[11px] text-[#8A80A8]`
  - `<YAxis>` with `text-[11px] text-[#8A80A8]`
  - `<Tooltip>` with custom content styled to match brand
  - No grid lines (`<CartesianGrid>` omitted or very subtle `stroke="#E2DEEC"` horizontal only)

### Sub-score averages table (below chart)
- 5 rows (one per score range: 80-100, 60-79, 40-59, 20-39, 0-19)
- Columns: Score Range, Count, Customers, Avg Fit, Avg Value, Avg Readiness, Avg State
- Compact table style per `Tables/_foundations.md` compact tier: `px-3 py-2`
- Header: `bg-[#F7F5FA]`, text: `text-[11px] font-semibold text-[#8A80A8] uppercase tracking-wider`

## Section 5 — Wild Card Signals

**Purpose:** Showcase the non-obvious insights from the analysis. The "aha" section.

### 3-column grid of insight cards

Each card contains:
- Title (e.g., "Outsourcing Spend vs Penetration")
- A brief insight sentence (e.g., "Districts paying >$1M to charters have 17% penetration — 4.5x the baseline")
- A Recharts horizontal `BarChart` showing the factor's buckets vs penetration rate
- Bar color: `#403770` (Plum)
- Labels: bucket name on Y-axis, penetration % on X-axis

**Card 1 — Outsourcing Behavior:**
Buckets: No payments, <$1M, $1-5M, $5-20M, $20M+
Metric: Penetration %

**Card 2 — Enrollment Trend:**
Buckets: Sharp Decline, Moderate Decline, Slight Decline, Stable, Growing
Metric: Penetration %

**Card 3 — Multi-Vendor Districts:**
Buckets: 0 vendors, 1, 2, 3, 4+
Metric: Avg revenue per district

Data for these charts is aggregated at render time from individual district records via `useMemo`. Each chart buckets the raw district fields (e.g., `charter_payments`, `enrollment_trend_3yr`, `vendor_count`) and computes penetration rates per bucket.

## File Structure

```
src/app/admin/icp-scoring/
  page.tsx                    — Main page, section layout, data loading, loading/error states
  types.ts                    — District interface (shared across all section components)
  hooks.ts                    — useDistrictScores() hook (fetch, loading, error state)
  components/
    HeroSection.tsx           — Section 1: methodology + stats
    TierDistribution.tsx      — Section 2: stacked bar + tier cards
    ScoreDistribution.tsx     — Section 3: Recharts histogram + sub-score table
    StateLandscape.tsx        — Section 4: sortable state table
    WildCardSignals.tsx       — Section 5: 3 insight cards with charts
    TopProspects.tsx          — Section 6: prospect cards
    DistrictExplorer.tsx      — Section 7: full table with filters + pagination
    shared.tsx                — Shared helpers: fmtNum, ScoreBar, SortArrow, tier/score color constants
```

Each section component receives the full `District[]` array and computes its own derived data via `useMemo`. No prop-drilling of computed stats — each section is self-contained. The `District` type is defined once in `types.ts` and imported by all components.

### Tier Classification Precedence

Tiers are applied top-down in the scoring script. A district is checked against Tier 1 criteria first (Fit >= 60 AND Value >= 60 AND Composite >= 60). If it doesn't qualify, it falls to Tier 2 (Composite >= 40), then Tier 3 (Composite >= 25), then Tier 4. No district appears in multiple tiers.

## Data Flow

```
page.tsx
  ├── fetch("/district_scores.json") → District[]
  ├── <HeroSection data={districts} />
  ├── <TierDistribution data={districts} />
  ├── <ScoreDistribution data={districts} />
  ├── <StateLandscape data={districts} />
  ├── <WildCardSignals data={districts} />
  ├── <TopProspects data={districts} />
  └── <DistrictExplorer data={districts} />
```

Each section receives the raw array. Section 7 (DistrictExplorer) manages its own filter/sort/pagination state internally.
