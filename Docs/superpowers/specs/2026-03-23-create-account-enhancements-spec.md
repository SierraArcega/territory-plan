# Feature Spec: Create Account Enhancements

**Date:** 2026-03-23
**Slug:** create-account-enhancements
**Branch:** worktree-create-account-enhancements

## Requirements

When resolving an unmatched opportunity via "Create new account", the user needs to:
1. Choose an account type (district, CMO, ESA/BOCES, cooperative, private school, state agency, university, organization, other)
2. For districts: manually enter 7-digit NCES LEAID
3. For non-districts: M000000-style code auto-generated by the backend
4. Search for and select a geocoded address (type-ahead via Nominatim)
5. The created account immediately resolves the opportunity

## Visual Design

**Approach:** Unified form with conditional fields inside the existing `DistrictSearchModal`.

- Account type dropdown at top (required, starts blank)
- Selecting "district" shows NCES LEAID field
- Selecting any non-district type shows "ID will be auto-generated" info text
- Account name field (pre-filled from opportunity.accountName)
- Address search field with type-ahead suggestions from Nominatim
- State dropdown + City field (auto-filled when address selected, manually editable)
- "Back" and "Create & Resolve" buttons

**Modal title** changes from "Create New District" to "Create New Account".
**Bottom link** changes from "Create new district" to "Create new account".

## Component Plan

### Modified components:
- `CreateDistrictForm` → replaced with `CreateAccountForm` in `page.tsx`
- `DistrictSearchModal` → title/link text updates

### New components (inline in page.tsx):
- `AddressSearchInput` — debounced Nominatim type-ahead with suggestion dropdown

### Existing components reused:
- State dropdown (already exists in form)
- Error alert pattern (already exists)
- Loading spinner pattern (already exists)

## Backend Design

### Existing endpoints (no changes needed):
- `POST /api/admin/districts` — creates district with manual LEAID
- `POST /api/accounts` — creates non-district account with auto-generated M-code + geocoding

### Minor enhancement:
- `POST /api/accounts` — add optional `lat`/`lng` pass-through to skip redundant geocoding when client already has coordinates from address search

## States

- **Loading:** Spinner in "Create & Resolve" button, fields disabled
- **Empty:** Account type not selected → only type dropdown visible, other fields hidden
- **Error:** Red alert banner with error message (existing pattern)
- **Address loading:** Small spinner in search field while Nominatim query is in-flight
- **Address empty:** "No results found" text in dropdown

## Out of Scope

- Editing existing accounts from this modal
- Bulk account creation
- Address validation beyond Nominatim geocoding
- Smart defaults based on opportunity reason tags
