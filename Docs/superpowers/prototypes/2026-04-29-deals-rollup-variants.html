Deals rollup — variants

Same week of mock data across all four prototypes. Sun $200K won + $369K lost (the original ambiguous $569K from the screenshot), plus assorted activity through the week. Each variant trades density vs. clarity differently.

A · Stacked rows, colored label cells current implementation

One row per kind. Tinted label cell + colored cell text. Highest clarity, biggest vertical footprint (~120px).

B · Stacked rows, slim color accent

Same five rows but the kind color is reduced to a 3px left edge and an icon. Quieter — looks more like the rest of the calendar.

C · Two summary rows: Net booked + Activity

Compress 5 rows into 2. Net = Won − Lost (green if positive, coral if negative, with breakdown on hover). Activity = count of new/progressed/closing, color-segmented.

D · Single dense row, in-cell breakdown

Back to one row, but each cell stacks the breakdown internally. Smallest vertical footprint, most visual noise per cell. Best for power users who want everything at once.

E · Two rows split by data type

One row stacks dollar bars (won + closing positive, lost negative). One row stacks count chips (new, progressed). Mixes visual encoding with text — heavier render but fastest scan.
